    "# Docstrings\n",
    "\n",
    "A [docstring](https://www.python.org/dev/peps/pep-0257/#id4) is a string embedded in a piece of code for the purpose of documenting it.\n",
    "\n",
    "Docstrings in BiFrEE are written in [NumPy style](). Why? Well, BiFrEE's main developers know there are other docstring styles, but the NumPy's one is clear, easy to be read, and widely extended.\n",
    "\n",
    "```{admonition} Note\n",
    ":class: note\n",
    "When BiFrEE's web documentation is compiled, [Sphinx](https://www.sphinx-doc.org/en/master/index.html) handles the docstrings with the assistance of [the Napoleon extension](https://www.sphinx-doc.org/en/master/usage/extensions/napoleon.html). Have a look to the [sphinx section in this guide](). \n",
    "```\n",
    "\n",
    "```{admonition} It is suggested...\n",
    ":class: tip, dropdown\n",
    "Although you can find some guidelines to write BiFrEE's docstrings below, if you are not familiar with the docstrings and the NumPy style you may also check the following webs:\n",
    "   - [PEP 257 -- Docstring Conventions](https://www.python.org/dev/peps/pep-0257/)\n",
    "   - [numpydoc's documentation](https://numpydoc.readthedocs.io/en/latest/format.html)\n",
    "   - [Example in numpydoc's documentation](https://numpydoc.readthedocs.io/en/latest/example.html)\n",
    "   - [Example in Napoleon's documentation](https://sphinxcontrib-napoleon.readthedocs.io/en/latest/example_numpy.html)\n",
    "```"

    "## Documenting a class\n",
    "\n",
    "```python\n",
    "\n",
    "class ClassName():\n",
    "    \"\"\"The summary line for a class docstring should fit on one line.\n",
    "    \n",
    "    If the class has public attributes, they may be documented here\n",
    "    in an ``Attributes`` section and follow the same formatting as a\n",
    "    function's ``Args`` section. Alternatively, attributes may be documented\n",
    "    inline with the attribute's declaration (see __init__ method below).\n",
    "\n",
    "    Properties created with the ``@property`` decorator should be documented\n",
    "    in the property's getter method.\n",
    "\n",
    "    Attributes\n",
    "    ----------\n",
    "    attr1 : str\n",
    "        Description of `attr1`.\n",
    "    attr2 : :obj:`int`, optional\n",
    "        Description of `attr2`.\n",
    "\n",
    "    \"\"\"\n",
    "\n",
    "    def __init__(a, b=14):\n",
    "         \"\"\"Example of docstring on the __init__ method.\n",
    "\n",
    "        The __init__ method may be documented in either the class level\n",
    "        docstring, or as a docstring on the __init__ method itself.\n",
    "\n",
    "        Either form is acceptable, but the two should not be mixed. Choose one\n",
    "        convention to document the __init__ method and be consistent with it.\n",
    "\n",
    "        Note\n",
    "        ----\n",
    "        Do not include the `self` parameter in the ``Parameters`` section.\n",
    "\n",
    "        Parameters\n",
    "        ----------\n",
    "        a : obj:`list` of :obj:`str`\n",
    "            Description of `A`. Multiple\n",
    "            lines are supported.\n",
    "        b : :obj:`int`, default=14\n",
    "            Description of `B`.\n",
    "\n",
    "        \"\"\"\n",
    "\n",
    "```\n",
    "\n",
    "\n",
    "```{admonition} See also\n",
    ":class: attention\n",
    "Before writting your own docstrings, have a look to [BiFrEE's code](https://github.com/uibcdf/BiFrEE/tree/main/sabueso) and the [API documentation](https://uibcdf.org/BiFrEE/api.html). There you can find useful examples on how the docstrings are written and rendered, and what are the usual conventions adopted in BiFrEE's docstrings.\n",
    "```"

    "## Documenting a method\n",
    "\n",
    "Some advices need to be followed to write a methods documentation. But first let's see an example of the docstring to document a method in BiFrEE's code:\n",
    "\n",
    "```python\n",
    "\n",
    "def method_name(a, b=True):\n",
    "    \"\"\"A short one line description without variable names or the method name\n",
    "\n",
    "    Paragraph with detailed explanation.\n",
    "\n",
    "    Parameters\n",
    "    ----------\n",
    "    a : int\n",
    "        Description of parameter `a`.\n",
    "    b : bool, default=True\n",
    "        Description of input argument `b` (the default is True, which implies ...).\n",
    "\n",
    "    Returns\n",
    "    -------\n",
    "    int\n",
    "        Description of the output int value.\n",
    "\n",
    "    Examples\n",
    "    --------\n",
    "    First example comment.\n",
    "    \n",
    "    >>> 2+2\n",
    "    4\n",
    "\n",
    "    Comment explaining the second example.\n",
    "\n",
    "    >>> import numpy as np\n",
    "    >>> np.add([[1, 2], [3, 4]],\n",
    "    ...        [[5, 6], [7, 8]])\n",
    "    array([[ 6,  8],\n",
    "           [10, 12]])\n",
    "\n",
    "    See Also\n",
    "    --------\n",
    "\n",
    "    :func:`bifree.get`, :func:`bifree.select`\n",
    "\n",
    "    Notes\n",
    "    -----\n",
    "\n",
    "    Section to include notes.\n",
    "\n",
    "    Todo\n",
    "    ----\n",
    "\n",
    "    Section to include a todo message.\n",
    "\n",
    "    Warning\n",
    "    -------\n",
    "\n",
    "    Section to include a warning message.\n",
    "    \n",
    "    \"\"\"\n",
    "    \n",
    "    pass\n",
    "```\n",
